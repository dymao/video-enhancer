# 使用说明

## 一、启动程序

```bash
python src/video_enhancer.py
```

程序启动后会显示主界面，包含以下功能区域：

- **B站视频链接输入框**：输入要下载的 B站视频 URL 或 B站短链
- **任务列表**：展示已添加的 B站视频任务，程序会按列表顺序逐个处理
- **输出目录选择**：选择视频保存位置（默认为 `output/` 目录）
- **输出格式选择**：选择视频输出格式（MP4、FLV、MKV）
- **清晰度增强选项**：勾选后启用视频增强功能
- **Cookies 文件选择**：用于下载需要登录的高清视频
- **开始下载按钮**：启动批量处理流程
- **取消当前按钮**：批量处理中会取消当前任务并继续下一个任务
- **取消全部按钮**：终止当前任务，并停止任务列表中的所有后续任务

## 二、下载视频

### 2.1 基本下载

1. 在"B站视频链接"输入框中输入 B站视频 URL
2. 点击"添加任务"按钮，将 URL 加入任务列表
3. 需要批量处理时，重复输入 URL 并继续点击"添加任务"
4. 如需调整任务，可删除选中任务或清空列表
5. 点击"开始下载"按钮后，程序会按任务列表顺序逐个下载、转换和增强
6. 流程区域会显示共有多少个任务、当前第几个任务进行中，以及当前任务执行到哪一步
7. 批量处理中点击"取消当前"会跳过当前任务，并继续处理任务列表中的下一个任务
8. 批量处理中点击"取消全部"会终止当前任务，并停止任务列表中的所有后续任务
9. 如果任务列表只有一个任务，"取消当前"和"取消全部"都会结束本次批量处理
10. 程序会自动将视频保存到指定的输出目录

### 2.2 支持的地址

程序当前仅支持以下 B站相关地址：

- `https://www.bilibili.com/video/...`
- `https://m.bilibili.com/video/...`
- `https://b23.tv/...`

下载前程序会先校验地址：

- 地址必须是 `http://` 或 `https://` 开头的网址
- 地址必须是 B站视频页或 B站短链
- 地址需要能正常访问，否则会弹框提示并停止下载
- 百度首页等普通网页地址会被识别为非视频地址，不会进入下载流程

### 2.3 高清视频下载

部分 B站高清视频需要登录后才能下载，需要配置 cookies 文件：

1. 在浏览器中登录 B站账号
2. 使用插件（如 EditThisCookie）导出 cookies 为 `cookies.txt`
3. 在程序中点击"浏览"选择 cookies 文件
4. 开始下载即可获取高清视频

## 三、视频增强

### 3.1 AI 超分辨率（推荐）

程序优先使用 Real-ESRGAN 进行 AI 超分辨率处理：

- 放大倍数：4 倍
- 模型：realesrgan-x4plus
- 特点：清晰度提升明显，细节丰富

**处理步骤**：
1. 将视频分解为单帧图片
2. 使用 AI 模型对每帧进行超分辨率处理
3. 将处理后的帧重新合成为视频

### 3.2 FFmpeg 增强（降级方案）

如果 Real-ESRGAN 不可用，程序会自动降级到 FFmpeg 增强：

- 高质量缩放（Lanczos 算法）
- 锐化滤镜
- 对比度增强
- 降噪处理
- 色彩调整

## 四、音视频同步

处理流程的最后一步是音视频同步：

- 检测原始视频帧率和时长
- 调整视频时间戳确保与音频同步
- 修复播放速度不一致问题

## 五、进度显示

程序提供详细的进度信息：

### 5.1 步骤状态

界面顶部显示 4 个步骤的状态：

- **下载**：蓝色圆圈表示进行中，绿色实心表示已完成
- **转换**：格式转换步骤
- **增强**：清晰度增强步骤
- **同步**：音视频同步步骤

### 5.2 进度条

- 显示当前步骤的进度百分比
- 支持两位小数精度

### 5.3 耗时显示

- 已耗时：显示从任务开始到当前的时间
- 剩余时间：根据当前进度预估的剩余时间

## 六、高级功能

### 6.1 帧缓存机制

程序会缓存已处理的视频帧：

- 如果检测到已提取的帧，跳过提取步骤
- 如果检测到已增强的帧，跳过 AI 处理步骤
- 支持断点续处理，节省重复处理时间

### 6.2 多线程优化

Real-ESRGAN 处理时使用多线程：

- 加载线程：4 个
- 处理线程：8 个
- 保存线程：4 个

### 6.3 取消任务

点击"取消"按钮可以终止当前任务：

- 终止正在运行的子进程
- 停止所有后续步骤
- 清理临时文件

## 七、常见问题

### Q1：下载失败，提示 SSL 错误？

A：这是因为 miniforge3 环境缺少 SSL 库。程序已自动使用系统 Python 运行 you-get，无需手动处理。

### Q2：Real-ESRGAN 不可用？

A：程序会自动检测工具和模型文件，如果缺失会尝试自动下载。也可以手动运行：

```bash
python tools/download_realesrgan.py
```

### Q3：处理速度很慢？

A：AI 超分辨率处理本身比较耗时，尤其是长视频。可以尝试：

- 确保使用 GPU 加速（需要 NVIDIA GPU）
- 减少视频分辨率或时长
- 使用 FFmpeg 增强方案（速度更快）

### Q4：视频处理后文件很大？

A：这是正常现象，高清视频文件较大。可以在代码中调整 CRF 参数（值越大文件越小，质量越低）。

### Q5：音视频不同步？

A：程序会自动进行音视频同步处理，如果问题仍然存在，可以尝试重新处理。

## 八、输出文件

处理完成后，输出文件保存在指定的输出目录：

- 原始下载文件：`原始文件名.mp4`
- 转换后文件：`原始文件名_processed.MP4`
- 增强后文件：`原始文件名_processed_enhanced.MP4`

## 九、临时文件

处理过程中会生成临时文件：

- `temp/temp_frames/`：提取的视频帧
- `temp/enhanced_frames/`：增强后的视频帧

处理完成后程序会自动清理临时文件。
